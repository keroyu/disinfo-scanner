## 第二天: 使用者管理

### ✓ 4. 檢視使用者列表

- [ ] 前往「使用者管理」頁面
- [ ] 檢視所有使用者列表
- [ ] 理解列表欄位:
  - [ ] 使用者 ID
  - [ ] 電子郵件
  - [ ] 目前角色
  - [ ] 電子郵件驗證狀態
  - [ ] 註冊日期

### ✓ 5. 練習搜尋與篩選

- [ ] 使用搜尋框搜尋特定用戶
- [ ] 使用角色篩選器
- [ ] 使用驗證狀態篩選器
- [ ] 使用日期範圍篩選器
- [ ] 測試分頁功能

### ✓ 6. 檢視使用者詳情

- [ ] 點擊任意使用者的「檢視」按鈕
- [ ] 檢視使用者完整資訊:
  - [ ] 基本資料
  - [ ] 角色與權限
  - [ ] API 配額使用情況 (如適用)
  - [ ] YouTube API 金鑰狀態
  - [ ] 身份驗證狀態

### ✓ 7. 練習角色變更 (測試用戶)

⚠️ **重要**: 請使用測試帳號練習,不要變更真實用戶的角色

- [ ] 建立測試用戶 (或請技術人員協助)
- [ ] 練習變更角色:
  - [ ] 一般會員 → 高級會員
  - [ ] 高級會員 → 網站編輯
- [ ] 確認角色變更立即生效
- [ ] 檢查稽核日誌是否記錄變更
- [ ] 嘗試變更自己的角色 (應顯示警告訊息)

## 第三天: 身份驗證審核

### ✓ 8. 理解身份驗證流程

- [ ] 閱讀《管理員使用指南》中的「身份驗證審核」章節
- [ ] 理解身份驗證的目的 (無限制 API 配額)
- [ ] 熟悉審核標準

### ✓ 9. 檢視驗證申請

- [ ] 前往「身份驗證審核」頁面
- [ ] 查看待審核申請列表
- [ ] 理解申請狀態:
  - [ ] 待審核 (Pending)
  - [ ] 已核准 (Approved)
  - [ ] 已拒絕 (Rejected)

### ✓ 10. 練習審核流程 (如有申請)

如果有待審核申請:

- [ ] 點擊「檢視」按鈕查看申請詳情
- [ ] 檢查驗證資訊的真實性
- [ ] 練習核准流程:
  - [ ] 點擊「核准」按鈕
  - [ ] 確認系統更新 API 配額為「無限制」
  - [ ] 檢查稽核日誌是否記錄
- [ ] 或練習拒絕流程:
  - [ ] 點擊「拒絕」按鈕
  - [ ] 輸入拒絕原因
  - [ ] 確認拒絕訊息已儲存

## 常見入職問題

### Q: 我完成所有步驟需要多久時間?

**A**: 大約 2-3 天,每天 2-3 小時。您可以按自己的節奏完成,但建議在第一週內完成所有步驟。

### Q: 如果我在入職過程中遇到技術問題怎麼辦?

**A**: 請立即聯繫技術支援: themustbig@gmail.com。不要嘗試自行修復系統問題。

### Q: 我可以跳過某些步驟嗎?

**A**: **不建議**。所有步驟都很重要,特別是安全性相關的步驟 (第 1、13、14、15 項)。如有特殊原因需要跳過,請與督導管理員討論。

### Q: 入職完成後,我需要多久檢查一次管理後台?

**A**: 建議每日至少檢查一次儀表板和待審核的身份驗證申請。稽核日誌建議每週檢查一次。

### Q: 如果我忘記如何執行某個操作怎麼辦?

**A**: 隨時參考《管理員使用指南》(/docs/admin-guide.md)。您也可以點擊管理後台右上角的幫助按鈕 (問號圖示) 查看功能說明。

### Q: 我需要記住所有使用者的權限嗎?

**A**: 不需要背誦,但建議熟悉權限矩陣 (在《管理員使用指南》中)。權限檢查時可以隨時查閱。
